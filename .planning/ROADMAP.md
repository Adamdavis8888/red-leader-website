# Roadmap: Red Leader Website

## Overview

Five-phase delivery of a lead-generation marketing website for Red Leader's emergency infrastructure rescue services. The roadmap progresses from foundation and core conversion path (booking capability) through trust building (case studies, team), content foundation (blog), and final optimization (performance, SEO). Each phase delivers verifiable capabilities that move technical buyers from awareness to engagement.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [x] **Phase 1: Foundation & Core Conversion** - Next.js foundation with homepage, services, contact, and booking
- [x] **Phase 2: Trust Building & Detail Pages** - Service details, team, about, trust signals, emergency messaging
- [ ] **Phase 3: Case Studies** - Portfolio of client success with filtering and quantified results
- [ ] **Phase 4: Blog** - Technical content with initial posts for thought leadership and SEO
- [ ] **Phase 5: Optimization & Polish** - Performance tuning, SEO completion, analytics, conversion optimization

## Phase Details

### Phase 1: Foundation & Core Conversion
**Goal**: Establish working Next.js architecture with immediate lead capture capability
**Depends on**: Nothing (first phase)
**Requirements**: PAGE-01, PAGE-02, PAGE-06, EMER-01, EMER-02, CONV-01, CONV-02, TECH-01, TECH-02, TECH-03, TECH-06
**Success Criteria** (what must be TRUE):
  1. Visitor sees clear value proposition above fold on homepage explaining emergency infrastructure rescue
  2. Visitor can view all service offerings on services overview page
  3. Visitor can book consultation via embedded Calendly from any page
  4. Visitor can submit contact form with name, email, company, and message
  5. 24/7 Emergency Response badge is visible on all pages
  6. Emergency hotline click-to-call appears in header on all pages
  7. Site renders correctly on mobile, tablet, and desktop breakpoints
  8. All pages served over HTTPS with valid SSL certificate
